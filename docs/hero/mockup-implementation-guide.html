Guide Technique — Implémentation Mockups & Interactions

Code vs Figma, comment ça marche, et quoi choisir pour ton portfolio

1. Browser Window Frame — Comment faire ?

3 approches possibles, avec ma recommandation pour ton cas.

✓ Approche A — CSS pur dans ton site portfolio

Tu wraps ton screenshot (<img>) dans une <div> qui dessine la barre macOS en CSS. Le screenshot est une simple image. La "chrome" (dots + URL bar) est du HTML/CSS autour.

→ Recommandé pour toi. Léger, responsive, et tu contrôles tout.

Approche B — Figma export

Tu crées le frame dans Figma (composant Device Mockup), tu places ton screenshot dedans, tu exportes le tout en PNG/WebP. Le browser frame fait partie de l'image.

→ Plus lourd (image plus grande), pas animable, mais simple si pas à l'aise en code.

Approche C — Librairie CSS (Flowbite, Preline)

Tu importes un composant tout fait type Flowbite Device Mockups ou Preline Devices. Copy-paste le HTML, mets ton image dedans.

→ Rapide mais ajoute une dépendance. OK pour prototyper.

Approche D — Outil mockup (DeviceFrames.com)

Tu upload ton screenshot, tu choisis un device, tu télécharges le résultat en PNG. Zéro code.

→ Utile pour des posts LinkedIn/Behance, pas pour un site web (image statique).

/* Le secret : overflow hidden + translateY au hover */ .browser-viewport { height: 300px; /* fenêtre visible fixe */ overflow: hidden; /* cache le reste */ } .screenshot { /* l'image est plus haute que la fenêtre */ transition: transform 4s cubic-bezier(0.25, 0.1, 0.25, 1); } .browser-frame:hover .screenshot { /* scroll vers le bas = on déplace l'image vers le haut */ transform: translateY(calc(-100% + 300px)); }
Comment ça marche : Ton screenshot est une image longue (la page entière). Le viewport a une hauteur fixe + overflow: hidden. Au hover, tu translateY l'image vers le haut. La transition CSS fait l'animation fluide. 3 lignes de CSS, zéro JS.

3. Vidéo d'interactions — Comment faire ?

Pour montrer des micro-interactions (hover states, transitions, animations de composants), il te faut de la vidéo. Voici les options.

✓ Option A — Vidéo MP4 dans le browser frame

Tu enregistres ton écran (Loom, OBS, macOS Cmd+Shift+5), tu exportes en MP4, et tu remplaces le <img> par un <video autoplay muted loop> dans le browser frame CSS.

→ Le meilleur rapport effort/impact. Autoplay + loop = le visiteur voit l'interaction sans cliquer.

Option B — GIF animé

Même enregistrement, mais converti en GIF (via Gifski, CloudConvert). Plus simple à intégrer mais bien plus lourd (5-10x le poids d'une vidéo MP4).

→ Seulement si tu ne peux pas utiliser <video> (email, Notion).

Option C — Lottie / animation CSS live

Tu recodes l'interaction directement en CSS/JS dans le portfolio. L'animation est "réelle" dans le browser.

→ Le plus impressionnant mais demande beaucoup de travail. Réserve pour 1-2 interactions clé.

Option D — Prototype Figma embarqué

Tu embarques un prototype Figma en <iframe>. Le visiteur peut interagir directement.

→ Cool mais lent au chargement, UX pas toujours fluide, et dépendant de Figma.

Démo — Vidéo MP4 dans un browser frame

Ce que tu dois faire

Le code est quasi identique au browser frame statique — tu remplaces juste <img> par <video>.

app.nod-factory.com/dashboard
Ta vidéo MP4 ici (autoplay, muted, loop) L'interaction se joue en boucle automatiquement
<!-- Remplace <img> par <video> — c'est tout --> <div class="browser-frame"> <div class="browser-chrome">...</div> <video src="./nod-interaction.mp4" autoplay <!-- joue automatiquement --> muted <!-- obligatoire pour autoplay --> loop <!-- boucle infinie --> playsinline <!-- pas de fullscreen sur mobile --> style="width: 100%; display: block;" ></video> </div>
Important : muted est obligatoire pour que autoplay fonctionne sur Chrome et Safari. Sans muted, le navigateur bloque la lecture auto. Et playsinline empêche iOS d'ouvrir la vidéo en plein écran.
Workflow recommandé :
1. Enregistre ton écran avec macOS Cmd+Shift+5 (gratuit) ou OBS (gratuit)
2. Coupe/crope avec QuickTime ou Kapwing (en ligne)
3. Compresse en MP4 H.264 avec Handbrake (gratuit) → vise < 2MB
4. Place le <video> dans ton browser frame CSS

4. Résumé — Quel format pour quel contenu ?

Ce que tu montres Format Dans quoi Effort
Un écran statique (dashboard, page) Image .webp Browser frame CSS ⚡ 10 min
Une page longue (scroll) Image longue .webp Browser frame + scroll-on-hover CSS ⚡ 15 min
Une micro-interaction (hover, transition) Vidéo .mp4 Browser frame + <video autoplay loop muted> ⚡ 20 min
Un flow complet (multi-écrans) Vidéo .mp4 Browser ou iPhone frame + video 30 min
Un prototype interactif Figma embed <iframe> 5 min (mais UX lente)
Version mobile Image ou vidéo iPhone frame CSS ⚡ 15 min
