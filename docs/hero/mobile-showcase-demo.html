Multi-Screen Mobile Showcase — 4 Techniques

Comment présenter plusieurs écrans mobile dans un portfolio — avec animation au hover

Figma export unique vs écrans individuels en front ?

Export Figma en 1 image

Tu arranges tes écrans dans Figma, tu exportes le frame entier en WebP.

✗ Pas d'animation possible (hover, scroll)

✗ Image lourde (8+ écrans = 1-3 MB)

✗ Pas responsive

✗ Re-export à chaque changement

→ OK pour Behance/Dribbble, pas pour ton site portfolio

Écrans individuels en front

Tu exportes chaque écran en WebP séparé, tu recrées le layout en CSS.

✓ Animation hover (colonnes qui scrollent)

✓ Léger (chaque image = 50-150 KB)

✓ Responsive natif

✓ Device frames CSS optionnels

→ C'est ce qu'il faut faire pour ton portfolio

Workflow : Dans Figma, sélectionne chaque écran → Export → WebP, 2x → tu obtiens home.webp, onboarding.webp, etc. Ensuite en front, tu les places dans des <img> à l'intérieur de ta grille CSS. Les device frames sont optionnels (divs CSS autour de chaque image).

/* Le layout : 3 colonnes flex, hauteur fixe, overflow hidden */ .showcase { display: flex; gap: 16px; height: 520px; /* fenêtre visible */ overflow: hidden; } .column { flex: 1; display: flex; flex-direction: column; gap: 16px; transition: transform 6s cubic-bezier(0.25, 0.1, 0.25, 1); } /* Offsets initiaux = les colonnes démarrent à des hauteurs différentes */ .col-1 { transform: translateY(0); } .col-2 { transform: translateY(-100px); } /* décalée vers le haut */ .col-3 { transform: translateY(-40px); } /* Au hover : certaines montent, d'autres descendent */ .showcase:hover .col-1 { transform: translateY(-180px); } .showcase:hover .col-2 { transform: translateY(80px); } .showcase:hover .col-3 { transform: translateY(-120px); }
Pourquoi sans device frame ici : Quand tu as 6+ écrans, les frames iPhone ajoutent du bruit visuel. Les écrans seuls avec un border-radius: 8px et un box-shadow suffisent — le cerveau comprend "c'est du mobile" grâce au ratio vertical. C'est le choix de ta capture Bfor.

Récap — Quelle technique pour quel cas

TechniqueNb écransDevice frames ?AnimationQuand l'utiliser
1. Colonnes sans frame 6-10+ Non (juste border-radius) Colonnes ↑↓ au hover Overview d'app mobile (comme ta capture Bfor)
2. Colonnes avec frame 3-6 Oui (iPhone CSS) Colonnes ↑↓ au hover Showcase premium, moins d'écrans
3. Hero + secondaires 1 hero + 3-6 Frame sur le hero seul Scale au hover sur les petits Quand 1 écran est plus important
4. Grid flat 4-8 Non Lift au hover Grid uniforme, scan rapide

Pour ton portfolio Bfor

1. Exporte chaque écran Figma individuellement en WebP 2x
2. Utilise la Technique 1 (colonnes sans frame) vu que tu as 8+ écrans
3. 3 colonnes, offsets différents, animation translateY au hover
4. Fond dark (#111827) pour faire ressortir les écrans clairs
5. Optionnel : ajoute 1 écran hero avec un iPhone frame pour le case study page (pas la card)
