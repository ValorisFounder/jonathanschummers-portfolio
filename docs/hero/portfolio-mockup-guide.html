Guide Mockups & Layouts Portfolio

Techniques CSS pour les device frames + 6 layouts les plus fréquents dans les portfolios design

Partie 1 — Device Mockups en CSS pur

5 techniques pour afficher tes screenshots dans des frames de devices. Tout est fait en CSS, aucune image externe requise. Remplace les fonds colorés par tes vrais screenshots.

1. Browser Window (macOS style)

CSS pur

Le classique. Parfait pour les apps web et dashboards. Les 3 dots macOS + barre d'URL donnent immédiatement le contexte "web app".

app.nod-factory.com/dashboard
Ton screenshot ici (16:10)
Quand l'utiliser : Apps web, dashboards, SaaS, landing pages. C'est le frame le plus commun dans les portfolios product design.

2. MacBook / Laptop

Premium

Plus immersif que le browser seul. Le device physique ajoute du réalisme et permet de montrer le produit "en situation".

Ton screenshot ici
Quand l'utiliser : Hero sections, projets premium, quand tu veux donner un côté "lifestyle" au produit.

3. iPhone / Mobile

CSS pur

Indispensable pour les projets mobile. Le dynamic island (notch) et le home indicator donnent le contexte iOS.

Ton screenshot mobile ici
Écran secondaire
Pro tip : Affiche 2-3 écrans côte à côte avec un léger scale/offset pour montrer un flow utilisateur complet.

4. Combo Desktop + Mobile (le plus impactant)

Recommandé

Le pattern le plus utilisé dans les portfolios top-tier : un browser desktop en arrière-plan avec un mobile qui overlap en bas à droite. Montre le responsive en un coup d'œil.

app.nod-factory.com
Desktop view
Mobile view
C'est LE layout à utiliser pour ton portfolio. Il montre que tu penses responsive, et le overlap crée de la profondeur visuelle. Ajoute un léger box-shadow sur le mobile pour le décoller du desktop.

5. Perspective / Vue 3D (hover pour animer)

CSS Transform

Un transform CSS en perspective donne un angle 3D isométrique. L'animation au hover "redresse" le mockup. Effet très premium, utilisé par Linear, Raycast, Vercel.

dashboard.example.com
Survole-moi pour voir l'animation
/* Perspective container */ .perspective-wrap { perspective: 1200px; } .mockup { transform: rotateY(-12deg) rotateX(4deg); transition: transform 0.4s ease; } .mockup:hover { transform: rotateY(-2deg) rotateX(1deg); }

6. Floating Animation

CSS Animation

Un léger mouvement de flottement donne vie au mockup sans être distrayant. Subtil mais efficace pour les hero sections.

nod-dashboard.app
Je flotte doucement
@keyframes float { 0%, 100% { transform: translateY(0); } 50% { transform: translateY(-8px); } } .floating { animation: float 3s ease-in-out infinite; }
